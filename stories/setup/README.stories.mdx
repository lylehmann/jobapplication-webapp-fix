# Job Application Manager - Storybook Setup

Welcome to the Job Application Manager design system documentation. This Storybook instance provides comprehensive documentation and interactive examples of all UI components, design tokens, and patterns used in the application.

## Quick Start

### Prerequisites

- Node.js 18+ 
- npm or yarn package manager

### Installation

1. **Clone the repository**
   \`\`\`bash
   git clone <repository-url>
   cd job-application-manager
   \`\`\`

2. **Install dependencies**
   \`\`\`bash
   npm install
   \`\`\`

3. **Start Storybook**
   \`\`\`bash
   npm run storybook
   \`\`\`

4. **Build Storybook for production**
   \`\`\`bash
   npm run build-storybook
   \`\`\`

## Design System Features

### 🎨 Color System
- **Semantic Colors**: Brand, success, warning, error, info
- **Employment Types**: Specific colors for job categories
- **Skill Levels**: Progressive color coding
- **Accessibility**: WCAG AA compliant contrast ratios

### 📝 Typography
- **Consistent Hierarchy**: H1-H6 with proper sizing
- **Font Weights**: Light to bold variations
- **Text Colors**: Primary, secondary, and muted variants

### 🧩 Components
- **Interactive Examples**: All components with live controls
- **Variants**: Multiple styles and states
- **Accessibility**: Built-in a11y features
- **Responsive**: Mobile-first design approach

## Development Workflow

### Adding New Components

1. **Create the component** in `components/ui/`
2. **Add component story** in `stories/components/`
3. **Document variants** and use cases
4. **Test accessibility** with built-in a11y addon

### Updating Design Tokens

1. **Modify** `tailwind.config.ts`
2. **Update** color/typography stories
3. **Test** component integration
4. **Document** changes in relevant stories

### Real-time Updates

Storybook automatically reloads when you make changes to:
- Component files
- Story files
- Tailwind configuration
- CSS files

## Available Addons

- **A11y**: Accessibility testing
